A Data model is a tree of elements organized in a hierarchy of schema --> tables --> fields.

A schema is a collection of named tables. Each table is a ordered collection of named fields. Each field belongs to a database type and has other properties.

The schema is modeled as a tree of SchemaElelemnt where each element is named and contained in another element.

Creating a schema with its element is defined by SchemaBuilder interface.

A schema can be created either in memory or from information from database. The concrete derivation of SchemaBuilder may access specific underlying database to build a schema.

A in-memory schema builder is available in this package that builds a schema from JSON descriptor.
