provides abstraction common to different flavors of database.

A connection profile holds meta-data about a connection. A connection conceptually operates on underlying database.

Connection interface does not describe any imperative function. Because imperative functions vary by underlying database (for example,. database in cloud versus on-premise). Hence, any grand generalization of database functions is avoided.

A concrete implementation is provided for CloudLite and can be used as a reference.

Typical usage is: 

    import oracle.nosql.model.connection.*;
    import oracle.nosql.model.cloud.connection.*;
    
	IConnectionProfile profile = new CloudsimConnctionProfile();
	profile.setUrl("http://localhost:8080/")  // properties depend on  
	       .setTenantId("TestTenant");        // specific profile
	
	IConnction c = profile.getConnection();
	
	CloudsimConnection cloudsimConnection = 
		c.unwrap(CloudsimConnection.class);

A CloudsimConnection is a thin wrapper over oracle.nosql.driver.NoSQLHandle .
